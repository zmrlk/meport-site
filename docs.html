What is Meport?

New to AI? No worries. You don't need to know anything technical. Meport does the hard part for you.

You probably use ChatGPT, Claude, or another AI assistant. But every time you start a conversation, the AI doesn't know anything about you. It gives you generic answers that could be for anyone.

Meport fixes that. It scans your files (CV, notes, documents), asks a few quick questions, and creates a profile of who you are — how you think, work, and communicate. Then it sends that profile to your AI tools so they actually understand you.

Think of it like setting up a new phone — you transfer your data once, and everything just works. Except instead of photos, you're transferring your identity to AI.

The result?

Instead of "Here are 10 generic tips," your AI says "Based on your preference for short answers and your work style, here's exactly what you need." Same AI, same question — completely different quality.

Meport is a CLI tool that creates deep AI personality profiles through pack-based profiling and exports them to 20+ platforms (ChatGPT, Claude, Cursor, Ollama) in their native format.

You use multiple AI tools. Each one treats you like a stranger. Meport creates a single profile with 80+ dimensions — communication style, thinking patterns, work habits, personal preferences — and exports it everywhere.

  • Pack-based profiling: 7 question packs covering identity, work, lifestyle, health, and more. Complete in 10-15 minutes.
  • System scanner: Reads your .gitconfig, project files, and AI configs to pre-fill answers automatically.
  • 20+ platform exports: ChatGPT, Claude, Cursor, Ollama, Copilot, Gemini, Windsurf, and more — each in their native format.
  • Rule-based output: "Never use emoji" instead of "User doesn't like emoji." Direct instructions, not descriptions.
  • 100% local: Your data never leaves your machine. No backend. No account.
  • Free forever: MIT license. Full features. No paywall.

Meport is an open-source CLI tool that generates rule-based AI personality profiles through pack-based profiling and compiles them to 20+ platform-native formats.

Terminal
npx meport profile            # start pack-based profiling (~10-15 min)
npx meport export chatgpt     # export to specific platform
npx meport export --all       # export to all 20+ platforms
npx meport sync               # auto-sync to file-based platforms

Architecture: TypeScript CLI (Commander.js). 7 question packs, 80+ dimensions across identity, communication, cognitive, work, personality, and more. 20+ rule-based compilers with platform-specific char limits. System scanner reads .gitconfig, project files, and AI configs to pre-fill answers. i18n: English + Polish. Fully offline, no backend.

  • Profile format: Local JSON at ~/.meport/
  • Export compilers: Rule-based compilation with per-platform char/token limits
  • 3-layer inference: Explicit (answers) → Inferred (behavioral signals) → Compound (rule combinations)
  • Key principle: Rules > Descriptions — each option has an export_rule that becomes a direct instruction

How profiling works

The full flow looks like this:

1

Greeting

Meport introduces itself and asks if you'd like to start with a file scan or go straight to questions.

2

File scan (optional)

Meport scans your files — CV, notes, configs, documents — and shows you what it found. You confirm or correct each finding.

3

Quick questions

Short multiple-choice questions fill in what the scan couldn't figure out. Each question takes 5-10 seconds.

4

Summary + corrections

Meport shows your complete profile. You review everything, fix anything that's wrong, and confirm when it looks right.

5

Export

Your answers become direct instructions for AI. For example, "Very direct" becomes the rule "Be extremely direct. No softening, no hedging." These instructions are formatted for each platform and exported.

Meport profiling has three layers of inference:

  • Layer 1 — Explicit: Your direct answers. You pick "Very direct" → rule: "Be extremely direct. No softening, no hedging."
  • Layer 2 — Inferred: Behavioral signals from how you answer (response patterns, choices across packs) generate additional rules.
  • Layer 3 — Compound: Rule combinations. If you selected both "concise" and "technical," a compound rule might emerge like "Give code examples instead of explanations."

The system scanner pre-fills answers from local files: .gitconfig (name, email, editor), project files (language, framework), and AI configs (.cursor/rules/, CLAUDE.md, Ollama modelfiles).

3-layer inference pipeline

  • Explicit (L1): Direct answer → option.export_rule. One-to-one mapping.
  • Inferred (L2): Behavioral signals from answer patterns across packs. Cross-referenced to generate additional rules.
  • Compound (L3): Rule combinations. Multiple L1/L2 rules that together imply a higher-order rule.

Key design principle: Rules > Descriptions. Every answer option has an export_rule field that becomes a direct instruction in the export. "Never use emoji" instead of "The user prefers not to receive emoji in responses."

How exports work

Once your profile is ready, you tell Meport which AI tool you use. It creates a set of rules in the exact format that tool expects.

For ChatGPT, it creates text you paste into Custom Instructions. For Cursor, it creates a file you put in your project folder. Each platform is different, but Meport handles the formatting for you.

Why it works so well Meport doesn't just describe you to AI. It gives AI direct instructions. Instead of saying "the user prefers short answers" (which AI often ignores), it says "Keep responses under 5 lines. Use bullet points." AI follows clear instructions much better than vague descriptions.

Meport has 20+ rule-based compilers, one per supported platform. Each compiler takes your profile dimensions and their export rules, prioritizes them by impact, and compiles them into the platform's native format within its character/token limits.

Terminal
meport export chatgpt          # ChatGPT Custom Instructions
meport export claude           # Claude Project Instructions (.md)
meport export cursor           # Cursor rules (.mdc)
meport export ollama           # Ollama Modelfile (SYSTEM directive)
meport export --all            # all 20+ platforms at once

Key design: exports are rule-based, not description-based. "Never use emoji" instead of "User doesn't like emoji." AI follows direct instructions more reliably than third-person descriptions.

20+ rule-based compilers. Each implements platform-specific formatting, character limits, and rule prioritization. The compiler selects the most impactful rules that fit within the platform's constraints.

Export format limits

ChatGPT1,500 chars × 2 fields, max 12 rules
Claude~2,000 tokens, Markdown with XML tags
Cursor12,000 chars, .mdc format, auto-synced to .cursor/rules/

Rule prioritization: rules are ranked by impact score (derived from dimension importance and cross-platform utility). The compiler fills the available space with the highest-impact rules first.

This copies the ChatGPT export to your clipboard so you can paste it into settings.

Sync auto-updates exports for file-based platforms:

Sync detects file-based platforms in the current directory and writes compiled exports directly:

Terminal

                               #   .cursor/rules/meport.mdc (if .cursor/ exists)

Sync is idempotent — running it multiple times produces the same result. It only writes when the compiled output differs from the existing file.

Rule engine

The rule engine is the core of Meport's export system. It transforms profile dimensions into platform-specific instruction sets.

Design principle: Rules > Descriptions

Every question option has an export_rule — a direct instruction, not a description. This is a deliberate design choice.

Rule vs. Description
# BAD (description-based, what competitors do):
"The user prefers concise responses and doesn't like emoji."

# GOOD (rule-based, what Meport does):
"Keep responses concise. Use bullet points. No filler."
"Never use emoji."

3-layer inference

  • Layer 1 — Explicit: Direct mapping from answer option to export_rule. One selection = one rule.
  • Layer 2 — Inferred: Behavioral signals from answer patterns. E.g., consistently picking short options → inferred rule about brevity.
  • Layer 3 — Compound: Multiple L1/L2 rules combined into higher-order rules. E.g., "concise" + "technical" + "direct" → "Give code examples instead of explanations. No preamble."

Compiler pipeline

Each compiler: collect rules → score by impact → select top N rules within char/token budget → format for platform → output.

Per-platform constraints
ChatGPT:  1,500 chars × 2 fields  |  max 12 rules
Claude:   ~2,000 tokens           |  Markdown + XML tags
Cursor:   12,000 chars            |  .mdc format
Ollama:   ~1,000 tokens           |  SYSTEM directive, max 8 rules

API

Meport can be used programmatically as an npm package:

TypeScript
import { loadProfile, compile } from 'meport';

// Load profile from default location (~/.meport/)
const profile = await loadProfile();

// Compile for specific platform
const chatgptExport = compile(profile, 'chatgpt');

// Access dimensions directly
const dims = profile.dimensions;
const verbosity = dims.find(d => d.dimension === 'verbosity_preference');
console.log(verbosity.export_rule);
// => "Keep responses concise. Use bullet points. No filler."

FAQ

Is Meport really free?

Yes. All features, all exports, all platforms. $0. Forever. MIT license.

Do I need an API key?

No. Meport uses rule-based profiling, not AI. Your answers map directly to export rules. No API keys, no internet connection, no AI provider needed.

Is my data safe?

Your data never leaves your computer. Meport has no servers, no accounts, no tracking. Your profile is stored locally. The only time data goes anywhere is when you manually paste or place an export into a platform.

Can I update my profile later?

Yes. Run meport update for a quick tune-up, or add new question packs with meport packs add. Run meport sync to push changes to your platforms.

How long does profiling take?

The full profile with all packs takes about 10-15 minutes. You can start with just the micro-setup pack (~2 minutes) and add more packs later. Use meport continue if you need to stop and resume.

What platforms are supported?

20+ platforms including ChatGPT, Claude, Cursor, Copilot, Gemini, Windsurf, Ollama, and more. Each has a dedicated compiler that formats your profile in the platform's native format.

What languages does Meport support?

English and Polish. The interface language can be set with --lang en or --lang pl.

What's the difference between export and sync?

export generates the output and shows it to you. sync automatically places the files where they need to go (e.g., .cursor/rules/, CLAUDE.md) and keeps them updated.
